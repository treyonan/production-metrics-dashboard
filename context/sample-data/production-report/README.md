## Real vs. synthetic rows in `sample.csv`

| Site ID | Row IDs  | Source    | Notes                                                               |
| ------- | -------- | --------- | ------------------------------------------------------------------- |
| `101`   | 40-101   | Real      | Big Canyon Quarry, Dolese Bros Co. Exported from SQL Server.        |
| `102`   | 201-262  | Synthetic | Generated by scaling site 101 values by 0.7 to exercise multi-site UI. |

The synthetic rows exist because the initial real export covers one site
only, which is not enough to demonstrate the enterprise multi-site
dashboard behavior. They are **demo data**, not a second real site.

### How the synthetic rows were generated

For each real row:

- `ID` shifted into the 201-262 range (clearly outside the real 40-101 range)
- `SITE_ID` changed from `101` to `102`
- `PROD_ID` prefix changed from `PR101_` to `PR102_`
- `PROD_DATE` / `DTM` unchanged (same date window as real data)
- Numeric values inside `PAYLOAD` scaled by `0.7`, with percentages
  (`Availability`, `Performance`) clamped to the 0-100 range and rounded
  to 1 decimal
- Operator names in `Site.Loader_Operator_*` replaced with fictional names
  (Jordan Martinez, Alex Chen) so real Dolese operators are not
  misattributed to synthetic activity

See `tasks/decisions/001-stack-and-source-boundary.md` for the decision
to go synthetic for multi-site demonstration during the POC phase.

When SQL integration lands, the SQL source will replace the CSV entirely
and synthetic data will no longer be in play.
